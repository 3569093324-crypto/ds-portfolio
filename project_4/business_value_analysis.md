# 业务价值分析：从模型指标到商业语言

> Day 51 | Phase 4: 机器学习核心能力

---

## 1. 业务场景定义

**场景**: 电商平台用户复购预测

**模型决策**: 预测用户在未来30天内是否会复购（二分类）

**业务应用**: 对预测为"不复购"的高风险用户，发放定向优惠券（成本 ¥10/张），激励其再次购买

**决策逻辑**:
- 预测"会复购" → 不发券（节省成本）
- 预测"不复购" → 发 ¥10 优惠券尝试挽回

---

## 2. 模型提升量化

### Baseline: 不使用模型

| 策略 | 做法 | 效果 |
|------|------|------|
| 全发券 | 给所有用户发 ¥10 券 | 成本=1000万用户 × ¥10 = ¥1亿，挽回复购的用户中有一半本来就会买（浪费） |
| 全不发券 | 不给任何用户发券 | 成本=0，但流失了那些"本来可以被挽回"的用户 |
| 随机发券 | 随机选50%发券 | 一半浪费 + 一半漏掉 |

### Model-based: 精准发券

使用 XGBoost 模型（AUC=0.96）：

| 指标 | 值 |
|------|-----|
| 月活跃用户 | 10,000,000 |
| 自然复购率 | 12% (1,200,000 人自然复购) |
| 模型 Precision@20% | 约 30% (Top 20% 高风险用户中 30% 确实会流失) |
| 模型 Recall | 约 70% (捕获了70%的真正流失用户) |
| 干预成功率 | 优惠券可挽回 15% 的流失用户 |

### 计算

```
不干预时:
  自然复购:     1,200,000 人
  潜在流失:     8,800,000 人
  → 其中可被挽回的: 8,800,000 × 15% = 1,320,000 人 (未挽回)

使用模型精准干预 (Top 20% = 2,000,000 人):
  模型识别高风险: 2,000,000 人
  其中真正会流失: 2,000,000 × 30% = 600,000 人
  被挽回的:       600,000 × 15% = 90,000 人/月
  发券成本:       2,000,000 × ¥10 = ¥20,000,000/月

假设用户平均LTV = ¥500:
  挽回收益 = 90,000 × ¥500 = ¥45,000,000/月
  ROI = (45,000,000 - 20,000,000) / 20,000,000 = 125%

对比"全发券":
  全发成本:     10,000,000 × ¥10 = ¥100,000,000/月
  挽回人数:     1,320,000 人 (但只有15%因为券)
  全发ROI:      (1,320,000 × ¥500 - 100M) / 100M = 560% → 但包含自然复购
  → 自然复购用户不需要券！模型省掉了 8,000,000 × ¥10 = ¥80,000,000 浪费
```

## 3. 量化商业影响

### 月度影响

| 指标 | 不使用模型（全发） | 使用模型（精准） | 差异 |
|------|-------------------|-----------------|------|
| 发券用户 | 10,000,000 | 2,000,000 | **-80%** |
| 发券成本 | ¥100,000,000 | ¥20,000,000 | **-¥80,000,000** |
| 挽回用户 | ~1,320,000 | ~90,000 | - |
| 挽回收益（增量） | ¥210,000,000 | ¥45,000,000 | - |
| 净利润（增量） | ¥110,000,000 | ¥25,000,000 | - |
| **ROI** | 110% | **125%** | **+15pp** |

> 注：模型的真正价值在于"省钱"——节省了对 800 万自然复购用户的不必要营销支出。

### 年度影响

- **年度节省营销成本**: ¥80,000,000 × 12 = **¥960,000,000 (9.6亿/年)**
- **年度增量挽回收益**: ¥25,000,000 × 12 = ¥300,000,000
- **总财务影响**: **约 ¥12.6 亿/年**

```
ROI = (收益 - 成本) / 成本

```

## 5. 局限性与风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| **数据漂移** | 用户行为模式变化，模型效果下降 | 每月监控 PSI (Population Stability Index)，>0.25 触发重训练 |
| **反馈循环** | 被标记为"高风险"的用户收到券 → 行为改变 → 下次预测偏差 | 保留 5% holdout 不发券作为对照组 |
| **公平性** | 模型可能对某些用户群有偏见 | 按 segment 分层评估模型效果，确保各组 Precision 差异 < 10% |
| **季节性** | 大促期间用户行为异常 | 模型输入中加入"是否大促期"特征 |
| **对抗行为** | 用户学会"假装流失"来获取优惠券 | 限制每人每月最多收 1 张券 |
