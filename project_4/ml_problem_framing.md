# ML 问题框架化：用户复购预测

> Day 35 | Phase 4: 机器学习核心能力

---

## a) 业务问题 → 数学问题

**业务问题**: 电商平台希望识别哪些用户会在未来30天内再次购买，以便进行精准营销（优惠券推送、个性化推荐），提升用户复购率和LTV。

**数学问题**: **二分类问题** — 给定用户历史行为特征，预测该用户在未来30天内是否会产生至少一笔新订单。

| 输入 | 输出 |
|------|------|
| 用户历史行为特征 (demographic + behavioral) | P(repurchase=1) ∈ [0, 1] |

**为什么是分类而不是回归？**
- 业务决策是二元的：发券/不发券，发推送/不发推送
- 需要的是"会不会买"而非"会花多少钱"
- 分类的概率输出可以直接用于排序（高概率用户优先触达）

---

## b) 目标变量定义

**y = 1**: 用户在观察窗口内（30天）至少产生一笔订单  
**y = 0**: 用户在观察窗口内（30天）零订单

**关键设计决策**:
- **观察窗口**: 30天 — 太短（7天）则正样本稀少，太长（90天）则预测周期过长
- **排除规则**: 观察期内注册的新用户（尚无"上次购买"概念，应单独建模）
- **单次 vs 多次**: 只要 ≥1 笔即标记为 1（复购定义明确）

**时间切分逻辑**:
```
[特征窗口: T-120 ~ T-30] → [间隔: T-30 ~ T] → [标签窗口: T ~ T+30]
                                   ↑
                              预测时间点 T (今天)
```
特征从过去120天提取，留30天gap避免数据泄漏，标签看未来30天。

## d) 评估指标选择

### 为什么不选 Accuracy？
复购用户是少数（~40%的用户），如果模型预测"所有人不复购"，accuracy 也能达到 60%，但模型完全无用。

### 选择 AUC-ROC + Precision@K

| 指标 | 用途 | 理由 |
|------|------|------|
| **AUC-ROC** | 模型整体区分能力 | 不受类别不平衡影响，适合排序问题 |
| **Precision@20%** | 业务价值 | 我们只对 Top 20% 用户发券，需要高精确率 |
| **Recall** | 辅助参考 | 确保覆盖足够多的潜在复购用户 |
| **F1-score** | 综合参考 | Precision 和 Recall 的调和平均 |

**业务约束**: 营销预算有限，每月只能触达 20% 的用户 → 需要 Precision@20% 高

### 基线 (Baseline)
- **随机预测**: AUC = 0.50
- **规则基线**: 用 RFM 中的 Recency（最近一次购买在30天内的用户）→ 预期 AUC ≈ 0.60-0.65
- **简单模型**: 逻辑回归 → 预期 AUC ≈ 0.70-0.75
- **目标**: XGBoost → 预期 AUC > 0.78

---

## e) 候选模型列表

| # | 模型 | 复杂度 | 优点 | 缺点 | 何时用 |
|---|------|--------|------|------|--------|
| 1 | **逻辑回归** | ★☆☆ | 可解释性强，系数直接对应特征重要性，部署简单 | 需要手动特征工程，无法捕捉非线性关系 | 需要解释"为什么"时（如向业务方汇报） |
| 2 | **随机森林** | ★★☆ | 自动处理非线性，特征重要性，对异常值鲁棒 | 模型较大，推理稍慢 | 特征较多且关系复杂时 |
| 3 | **XGBoost** | ★★★ | 性能通常最好，内置正则化，处理缺失值 | 超参数多，调参复杂，可解释性差 | 追求最佳预测效果时 |
| 4 | **LightGBM** | ★★★ | 比XGBoost更快，leaf-wise生长，支持类别特征 | 小数据量易过拟合 | 大数据量场景 |

**推荐策略**: 
1. 先用逻辑回归建立可解释的基线
2. 用 XGBoost 提升效果
3. 如果 XGBoost 提升不明显（<2% AUC），保留逻辑回归（简单 + 可解释）

## f) 基线 (Baseline)

### Baseline 1: 随机猜测
- AUC = 0.50 (任何模型必须高于此)

### Baseline 2: 启发式规则 — "最近30天有购买的用户会复购"
- 基于 RFM 的 Recency：如果 `last_purchase_days <= 30` → 预测复购
- 预期：Precision ≈ 50%, Recall ≈ 40%

### Baseline 3: 逻辑回归（仅用3个特征）
- 仅用 Recency + Frequency + Monetary (RFM三特征)
- 预期：AUC ≈ 0.65-0.70

**任何复杂模型必须先超越 Baseline 3 才有意义。**

## 🤔 反思：如果模型效果不好，可能是数据问题还是模型问题？

### 更可能是数据问题的信号
- 训练集和测试集分布不一致（时间切分问题：用过去预测未来时，用户行为模式可能变化）
- 特征缺乏区分力（如所有用户都在最近30天买过 → Recency特征失效）
- 正负样本极度不平衡且使用了不适合的指标
- 标签噪声（如促销活动导致大量"假复购"）

### 更可能是模型问题的信号
- 不同模型表现差异巨大（如LR AUC=0.65, XGBoost=0.78 → LR欠拟合）
- 训练集AUC远高于测试集AUC（过拟合 → 需要正则化/减少特征/增加数据）
- 增加更多数据后效果不提升（模型容量不够）

### 诊断优先级
```
1. 检查数据切分是否正确 (时间泄漏?)     ← 最常见问题
2. 分析特征重要性 (哪些特征在起作用?)   ← 指导特征工程
3. 看学习曲线 (是否需要更多数据?)        ← 判断过拟合/欠拟合
4. 尝试不同模型复杂度 (LR → RF → XGB)   ← 判断是否是模型问题
```
