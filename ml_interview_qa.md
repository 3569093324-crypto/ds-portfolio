## 1. Bias-Variance Tradeoff？

**Bias（偏差）**: 模型预测与真实值的系统性偏离 → 欠拟合（模型太简单）
**Variance（方差）**: 模型对训练数据波动的敏感性 → 过拟合（模型太复杂）

**Tradeoff**: 简单模型（线性回归）→ 高Bias低Variance；复杂模型（深度决策树）→ 低Bias高Variance。目标是在两者之间找到最优平衡→最小化总误差。

---

## 2. 过拟合怎么检测？怎么解决？（4种方法）

**检测**: 训练集性能远好于测试集/验证集 → Train-Test Gap大。

**解决**:
1. **增加数据**: 更多样本，数据增强
2. **正则化**: L1/L2惩罚大权重
3. **简化模型**: 减少特征/降低树的深度
4. **早停 (Early Stopping)**: 验证集loss不再下降就停止训练
5. **集成**: Bagging（Random Forest）通过平均降低方差

## 3. L1 vs L2 正则化？

| | L1 (Lasso) | L2 (Ridge) |
|---|---|---|
| 惩罚项 | λΣ\|w\| | λΣw² |
| 效果 | 产生稀疏解（权重=0） | 权重收缩但不为0 |
| 特征选择 | ✅ 自动特征选择 | ❌ |
| 适用 | 高维特征、需要稀疏性 | 处理共线性 |

**L1能做特征选择的原因**: L1的约束域是菱形，最优解容易落在坐标轴上→某些权重恰好为0。

## 4. 逻辑回归的损失函数？为什么不用MSE？

**Log Loss (Cross-Entropy)**: J(w) = -(1/n)Σ[y·log(p) + (1-y)·log(1-p)]

**为什么不用MSE**: 当p接近0或1时，sigmoid函数的梯度趋近于0（梯度消失），导致参数更新极慢。Log Loss恰好在p接近边界时梯度大→加速收敛。

## 5. SVM的核心思想？核函数的作用？

**核心思想**: 找一个最大化类别间隔（margin）的超平面来分离数据。只有支持向量（边界上的点）影响决策边界。

**核函数**: 将数据从低维空间映射到高维空间，使原本线性不可分的数据在高维变得线性可分。常用：RBF（高斯核）、多项式核。

## 6. 决策树分裂标准？Gini vs 信息增益？

- **Gini Impurity**: G = 1 - Σp_i²。计算更快，sklearn默认。
- **Entropy (信息增益)**: H = -Σp_i·log(p_i)。偏好产生更平衡的分裂。
- 实践中差异不大，Gini稍快。

---

## 7. 随机森林的两个随机性？

1. **样本随机**: Bootstrap采样→每棵树用不同样本子集训练
2. **特征随机**: 每次分裂只考虑√d个特征（而非全部d个）
→ 双重随机→树之间相关性低→方差降低→泛化能力增强

## 8. XGBoost vs GBDT？

| | GBDT | XGBoost |
|---|------|---------|
| 导数 | 仅一阶导数 | **二阶导数**（更快更准） |
| 正则化 | 无 | L1+L2内置正则化 |
| 列采样 | 无 | 支持（类似RF特性） |
| 缺失值 | 需预处理 | **自动处理缺失** |
| 并行 | 无 | 特征级别并行 |

**核心改进**: 二阶导数+正则化+自动缺失值处理

## 9. Bagging vs Boosting？

| | Bagging (RF) | Boosting (XGBoost) |
|---|---|---|
| 训练 | 并行（各树独立） | 串行（后一棵纠正前一棵错误） |
| 目标 | 降低方差 | 降低偏差 |
| 样本 | Bootstrap独立抽样 | 根据上一轮误差加权 |
| 适合 | 高方差模型（深树） | 高偏差模型（浅树） |

## 10. 交叉验证为什么重要？K怎么选？

**为什么**: 单次train/test split可能因运气好而得到虚高分数→CV提供更稳健的评估。

**K怎么选**:
- K=5或10最常见，平衡偏差和计算量
- K小→偏倚大但快；K大→偏倚小但慢
- K=n（留一法）→几乎无偏但计算量最大

## 11. 不平衡数据处理（4+方法）？

1. **重采样**: SMOTE过采样少数类 / RandomUnderSampler降采样多数类
2. **类别权重**: class_weight='balanced' → 少数类错误代价更高
3. **阈值调整**: 移动分类阈值（如从0.5降到0.3以提高Recall）
4. **评估指标**: 不用Accuracy，用PR-AUC / F1 / ROC-AUC
5. **集成方法**: BalancedRandomForest, EasyEnsemble

## 12. AUC-ROC vs PR曲线？

| | ROC | PR |
|---|---|---|
| X轴 | FPR | Recall |
| Y轴 | TPR | Precision |
| 适合 | 类别平衡时 | **类别不平衡时** |
| Baseline | 对角线(AUC=0.5) | 正样本比例 |

**选择**: 不平衡数据→PR曲线（ROC可能过于乐观）。

## 13. 特征选择方法？

1. **Filter（过滤法）**: 统计检验（卡方、相关系数、互信息）→ 不依赖模型，快
2. **Wrapper（包装法）**: RFE（递归消除）、前向/后向选择 → 依赖模型，慢但准
3. **Embedded（嵌入法）**: L1正则化、树模型特征重要性 → 模型自带

## 15. 标准化 vs 归一化？

- **标准化 (Z-score)**: (x-μ)/σ → 均值为0，标准差为1。适合：大多数ML（假设正态）
- **归一化 (MinMax)**: (x-min)/(max-min) → 缩放到[0,1]。适合：需要固定范围的（神经网络、图像）

**注意**: 标准化/归一化必须在train/test split之后，只用训练集的参数transform测试集！

## 16. 什么是数据泄露？举例。

**定义**: 训练过程中使用了不应获取的信息（测试集、未来信息），导致评估虚高。

**例子**: 
1. 先对整个数据集做标准化再切分→scaler看到了测试集的均值/方差
2. 用未来的价格预测过去的趋势（时间序列）
3. 特征工程中用了目标变量的信息（如：用y的均值做填充）

## 17. PCA原理？什么时候用？

**原理**: 通过SVD找到数据方差最大的方向（主成分），将高维数据投影到低维子空间，保留尽可能多的方差。

**何时用**: 高维数据降维（>1000维）→可视化、去噪、加速训练。**注意**：PCA后特征失去可解释性。

## 18. 模型部署后效果变差，可能原因？

1. **数据漂移**: 用户行为/业务环境改变→训练集分布≠线上分布
2. **概念漂移**: 输入和输出的关系本身变了
3. **反馈循环**: 模型预测影响用户行为→行为数据又喂给模型
4. **训练-服务偏差**: 特征工程在生产环境实现不一致
5. **季节性**: 模型在特定时期训练，无法泛化到其他时期

**解决**: 监控PSI（群体稳定性指数）、定期重训练、A/B测试验证

## 19. Target Encoding怎么做？有什么风险？

**做法**: 用目标变量的统计量（均值）替换分类变量的每个类别。
例: city='NYC' → 用NYC用户的平均转化率替换。

**风险**: 
1. **过拟合**: 罕见类别只有少量样本→均值不可靠
2. **数据泄露**: 如果用全局均值（未做CV），训练集信息泄露到验证集

**缓解**: 
- 平滑: encoded_value = (n*cat_mean + m*global_mean) / (n+m)
- CV Target Encoding: 用其他fold的均值编码当前fold

## 20. SHAP值如何计算？

**SHAP (SHapley Additive exPlanations)**: 基于博弈论的Shapley值，将模型预测分解为每个特征的贡献。

**核心思想**: 对某个样本，计算每个特征在所有可能的特征组合中的边际贡献的平均值。

**公式**: φ_i = Σ_{S⊆N\\{i}} [|S|!·(|N|-|S|-1)!/|N|!] · [f(S∪{i}) - f(S)]

**性质**: 
- 加和性: base_value + ΣSHAP_values = prediction
- 一致性: 特征越重要，SHAP值越大
- 可解释: 正SHAP→推向正预测，负SHAP→推向负预测
